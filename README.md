# SaleeM v2.4

تطبيق متخصص في تحليل الذهب `XAUUSD` وتنفيذ السيناريو على فريم خمس دقائق، مع جلب بيانات `M5` و`M15` و`H1` و`H4` تلقائيًا من Twelve Data.

## طريقة العمل

1. يرفع المستخدم صورة شارت M5.
2. يجلب التطبيق بيانات الذهب الفعلية للفريمات الأربعة من Twelve Data.
3. يستخدم H4 للاتجاه الرئيسي، وH1 لبنية السوق، وM15 لمنطقة التفعيل، وM5 لتوقيت الدخول.
4. يستخدم صورة المستخدم مرجعًا نهائيًا لسعر الدخول والوقف والأهداف، لأن سعر مزود البيانات قد يختلف قليلًا عن سعر الوسيط.
5. يعيد رسم نافذة مرنة من شموع M5 المتاحة في صورة عمودية واضحة بدقة 1080×1920.

## التخزين المؤقت

يحفظ التطبيق البيانات في ملف JSON واحد ويحدث فقط الفريم المنتهي:

- M5: كل 4 دقائق.
- M15: كل 14 دقيقة.
- H1: كل 55 دقيقة.
- H4: كل 4 ساعات.

عند تعطل Twelve Data مؤقتًا، يستخدم التطبيق آخر نسخة محفوظة للفريم إذا كانت موجودة بدل إيقاف التحليل بالكامل.

المسار الافتراضي:

```text
/tmp/saleem_market_data_cache.json
```

هذا المسار قد يُحذف عند إعادة تشغيل أو نشر Railway. للتخزين الدائم، أضف Railway Volume بمسار تركيب `/data` ثم استخدم:

```env
MARKET_DATA_CACHE_PATH=/data/saleem_market_data_cache.json
```

## متغيرات Railway

انسخ القيم الموجودة في `.env.example` إلى صفحة **Variables** داخل Railway. المتغيران الإلزاميان هما:

```env
OPENAI_API_KEY=...
TWELVE_DATA_API_KEY=...
```

لا تضع المفاتيح الحقيقية داخل ملفات GitHub.

## التشغيل المحلي

```bash
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## القالب النهائي

- العنوان: تحليل SaleeM - XAUUSD - M5.
- شموع السوق المتاحة في صورة عمودية واضحة دون اشتراط عدد ثابت.
- أقرب دعمين ومقاومتين، وسماكة الخط حسب القوة.
- سيناريو واحد، دخول أو دخول محتمل، وقف بنيوي، وTP1 وTP2 وTP3.
- يظهر ملخص اتجاه `H4` و`H1` و`M15` داخل ملاحظات الصورة.
- المرجع الأعلى هو `SALEEM_FINAL_SPEC.md`.

## التحقق بعد النشر

افتح المسار `/health` بعد النشر. يجب أن تظهر:

```json
{
  "openai_configured": true,
  "twelve_data_configured": true
}
```

هذه القيم تؤكد وجود المتغيرات فقط ولا تعرض المفاتيح السرية.


## معالجة خطأ OpenAI 429

الإعداد الافتراضي يستخدم `gpt-4.1-mini` ويرسل عددًا مخفضًا من شموع السوق لتجنب حد الرموز في الدقيقة.
المتغيرات المقترحة في Railway:

```env
OPENAI_MODEL=gpt-4.1-mini
OPENAI_MAX_OUTPUT_TOKENS=5000
OPENAI_RETRIES=2
MARKET_CONTEXT_CANDLES=40
```

يعرض التطبيق الآن سبب 429 بشكل أوضح، ويسجل `x-request-id` في Logs دون إظهار المفاتيح السرية.

## إعداد نافذة الرسم

- `CHART_CANDLE_COUNT`: عدد شموع M5 المستخدمة في صورة النتيجة، والقيمة الافتراضية `30`، والمجال المدعوم من `12` إلى `48`.
- لا يشترط أن تحتوي صورة المستخدم على عدد محدد من الشموع؛ المطلوب أن يكون السعر الحالي ومحور السعر واضحين.
- شموع الرسم والتوقيتات والمناطق الفنية مشتقة من Twelve Data، بينما السعر الحالي يُقرأ من الصورة المرفوعة.
